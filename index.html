    Introduction

    Welcome to the Phonolgy API! You can use our API to access Phonology API endpoints, which can get information on various users and patient in our database using node.js, express.js and mongoDB.

    We have language bindings in JavaScript! You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Run the node server

    To use the API, you need to have an instance running locally on your computer.

    To run the server, use this code:

    npm install
nodemon server.js

    Users

    Post practitioner

    const app = express();

app.get('/create_practitioner', (req, res) => {}

    The above command returns JSON structured like this:

    {
  "email": "azdf",
  "password": "test",
  "group": "medecin",
  "name": "azer",
  "_id": "5cd4aca4add38946a5aef105"
}

    This endpoint create a user.

    HTTP Request

    POST http://localhost:8000/create_practitioner

    Query Parameters

    Parameter Description
    email email of the user, mandatory
    password password of the user, mandatory
    group group of the user, mandatory
    name name of the user, mandatory
    MongoDB create an unique _id for us

    Delete a Specific User

    const app = express();

app.delete('/update_user_by_id/:id', (req, res) => {}

    HTTP Request

    PUT http://localhost:8000/update_user_by_id/:id

    URL Parameters

    Parameter Description
    id The ID of the user to delete
    email email of the user (in body request)
    password password of the user (in body request)
    name name of the user (in body request)
    group group of the user (in body request)

    The above command returns html answer (for now) like this:

    User 'id' deleted

    This endpoint delete a specific user.

    HTTP Request

    DELETE http://localhost:8000/delete_user_by_id/:id

    URL Parameters

    Parameter Description
    id The ID of the user to delete

    Errors

    The Phonology API uses the following error codes:

    Error Code Meaning
    200 Good request -- Your request in valid.
    400 Bad Request -- Your request is invalid.
    403 Forbidden -- The user requested is hidden for administrators only.
    404 Not Found -- The specified user could not be found.
    405 Method Not Allowed -- You tried to access a user with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The user requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many users! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
